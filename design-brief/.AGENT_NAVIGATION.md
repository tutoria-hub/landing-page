## File Roles

### .AGENT_NAVIGATION.md (This File)
Agent instruction manual. Read first when user mentions landing page.

### DESIGN_BRIEF.md
Master vision. Contains global specs (typography, colors, spacing, buttons). Read for context.

### _global/
Shared assets. Logo, design system docs, original wireframe.

### Section Folders (01-hero/ through 06-founder/)
Isolated context per section:
- **README.md**: Design specs (created iteratively with user feedback)
- **assets/**: User-added images/mockups
- **notes.md**: User's scratch space (YOU READ, DON'T WRITE unless asked)

## Basic Workflows

### User Asks About Specs

**Example**: "What's the hero headline size?"

**Process**:
1. Identify section: "hero" → `01-hero/`
2. Read `01-hero/README.md`
3. Find component specs
4. Return: "96px desktop, 48px mobile (Nunito Bold 700). See 01-hero/README.md."

### User Adds Asset

**Example**: "I added a mockup to hero"

**Process**:
1. Check `01-hero/assets/` for files
2. Read `01-hero/notes.md` for context
3. Ask: "Should I update README based on mockup? What changes?"

### User Requests New Section

**Example**: "Create video demo section"

**Process**:
1. Verify hero validated first
2. Read `01-hero/README.md` as pattern template
3. Read `DESIGN_BRIEF.md` for global specs
4. Create `02-video-demo/README.md`
5. Present for feedback → iterate

---

## Iterative Creation Process

**CRITICAL**: Sections created ONE AT A TIME with user feedback.

**Flow**:
1. Hero exists (example)
2. User validates hero → feedback → iterate
3. User: "now do video demo"
4. Create `02-video-demo/README.md`
5. User validates → feedback → iterate
6. Repeat for remaining sections

**DO NOT** create all sections upfront.

## Global Design Reference

See `DESIGN_BRIEF.md` for complete specs:

**Typography**: Nunito Bold 700 (headers), Regular 400 (body)
**Colors**: #30A46C (green), #F7F5ED (cream), #1A1A1A (text)
**Spacing**: 8px grid (96px sections, 24px padding)
**Buttons**: Pill (border-radius: 9999px), 20px/48px padding

2. **Create sections iteratively** (one at a time)
3. **DESIGN_BRIEF.md is authority** for global specs
4. **Don't write to notes.md** (user-owned)
5. **One section at a time** (validate before next)
